# 查询中心旧版员工选择器复刻设计

## 目标

将 React 前端里当前简化版的员工选择器替换为接近旧版模板行为的三栏弹窗员工选择器，覆盖当前已经接入 `EmployeePicker` 的页面：

- 查询中心页
- 汇总下载页
- 管理人员考勤修正页

本次仅调整前端组件与交互，不修改后端接口、查询参数格式或页面业务提交逻辑。

## 范围

### 包含

- 将 `frontend/src/components/query/EmployeePicker.tsx` 从“搜索 + 多选下拉框”改为“触发器 + 弹窗”模式
- 在 React 组件内复刻旧版员工选择器的核心布局与交互：
  - 左侧部门树
  - 中间候选员工列表
  - 右侧已选员工列表
  - 搜索
  - 全选当前可见员工
  - 清空
  - 取消 / 确定
- 让当前引用 `EmployeePicker` 的 React 页面继续使用原有组件接口
- 补充前端测试，覆盖关键交互路径

### 不包含

- 后端 API 变更
- 异步远程搜索
- 虚拟滚动
- 对旧模板页面或旧版原生 JS 选择器的重构
- 顺手修改其他选择器（如部门选择器）

## 方案选择

### 方案 A：保留现有组件接口，重写为 React 版旧式弹窗选择器

优点：

- 影响面最小
- 页面业务逻辑基本不需要改
- 更符合当前 React 前端结构

缺点：

- 需要在 React 中重新实现旧版交互，而不是直接复用旧模板 JS

### 方案 B：尽量直接搬旧模板和旧版 DOM 脚本

优点：

- 视觉和交互最容易贴近旧版

缺点：

- 会引入大量命令式 DOM 操作
- 与 React 状态流不一致
- 维护成本高

### 方案 C：只复刻视觉，不复刻完整交互

优点：

- 实现更快

缺点：

- 不能满足“复刻旧版员工选择器”的目标

### 结论

采用方案 A：保留 `EmployeePicker` 外部接口，在内部实现 React 版旧式三栏弹窗选择器。

## 组件设计

`EmployeePicker` 继续保留以下输入输出：

- `employees`
- `filterMode`
- `label`
- `selectedIds`
- `onChange`

组件内部改为两层状态：

- 外部受控状态：`selectedIds`
- 弹窗内部草稿状态：当前弹窗中的临时选择结果

交互规则：

- 点击触发器打开弹窗时，用外部 `selectedIds` 初始化草稿状态
- 用户在弹窗中的勾选、搜索、部门切换、清空，只影响草稿状态
- 点击“确定”时，把草稿状态写回 `onChange`
- 点击“取消”或关闭弹窗时，丢弃草稿状态

这样可以保持旧版“先选、再确认”的操作模型，同时避免页面业务代码改动。

## 交互设计

复刻旧版核心交互，但以 React 方式实现：

### 1. 触发区

组件默认展示：

- 字段标题
- 只读摘要框或按钮
- 已选人数提示

摘要优先显示：

- 未选择时：提示默认查询可见范围
- 已选择时：显示前若干个员工名或工号摘要，并附已选人数

### 2. 左侧部门树

根据 `employees` 中可用的部门信息生成本地部门分组树。

支持：

- “全部部门”根节点
- 按部门筛选候选员工

不要求完整复刻旧版所有树行为，只保留当前查询场景需要的部门过滤能力。

### 3. 中间候选区

候选区包含：

- 搜索框
- “全选当前可见员工”复选框
- 员工候选列表

过滤规则按顺序叠加：

- `filterMode`
- 当前选中的部门
- 搜索关键字（工号 / 姓名 / 部门名）

### 4. 右侧已选区

显示当前草稿中的已选人员：

- 员工工号
- 员工姓名
- 部门名

支持：

- 单条移除
- 一键清空

### 5. 底部动作

- `取消`：关闭弹窗，不提交
- `确定`：提交草稿选择

## 页面落点

本次通过统一替换 `EmployeePicker`，自动覆盖这些 React 页面：

- `frontend/src/pages/query/QueryPage.tsx`
- `frontend/src/pages/query/SummaryDownloadPage.tsx`
- `frontend/src/pages/admin/ManagerAttendanceOverridesPage.tsx`

这些页面不需要修改查询参数结构，只需要继续传入当前的 `selectedIds` 与 `onChange`。

## 样式策略

样式延续 `frontend/src/styles/legacy-ui.css` 的旧版产品壳风格。

新增样式包括：

- 触发区
- 弹窗遮罩 / 弹窗主体
- 三栏布局
- 部门树节点
- 候选列表项
- 已选列表项
- 底部操作区

视觉目标：

- 接近旧版模板结构
- 保持当前 React 前端的旧版主题色和边框体系
- 在桌面端优先保证可用性

移动端只要求不坏，不在本次追求完整移动端复刻。

## 测试策略

新增或调整前端测试，覆盖以下行为：

- 打开员工选择器弹窗
- 初始已选项正确回显
- 搜索过滤候选员工
- 切换部门后候选结果变化
- 全选当前可见员工
- 单条移除已选人员
- 清空已选人员
- 点击取消不提交
- 点击确定后调用 `onChange`

测试重点是行为一致性，不要求快照式精确还原旧模板 DOM。

## 风险与约束

### 风险

- 旧版模板依赖后端模板上下文和命令式 DOM 行为，React 复刻时不能逐行照搬
- 当前 `employees` 数据结构若部门层级信息不完整，部门树只能做本地近似复刻
- 页面较多共用同一组件，必须防止某页行为修好、另一页回归

### 应对

- 保持组件外部接口不变
- 优先实现查询场景真正依赖的旧版行为
- 用组件级或页面级测试覆盖共用路径

## 成功标准

满足以下条件即认为本次完成：

- React 查询相关页面不再使用简化多选下拉框
- 员工选择入口改为旧版风格的三栏弹窗
- 查询中心、汇总下载、管理人员考勤修正页都能正常选择、取消、清空、确认员工
- 不需要修改后端接口或页面查询参数
- 相关前端测试通过
